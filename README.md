# Проект: Контейнеризация на Web Application с Docker Compose

Този проект демонстрира успешна контейнеризация на двукомпонентно уеб приложение, състоящо се от Node.js/Express бекенд сървър и MongoDB база данни. Проектът е конфигуриран да се стартира и управлява изцяло чрез Docker Compose.

---

## 📂 Структура на проекта

Проектът е организиран в следната файлова структура:
*   `server.js` — Основният изходен код на уеб сървъра (Бекенд архитектура).
*   `package.json` — Конфигурационен файл с метаданни, скриптове и зависимости на Node.js проекта.
*   `Dockerfile` — Спецификация и инструкции за изграждане на Docker образа на нашето уеб приложение.
*   `compose.yml` — Оркестрационен файл, който дефинира, свързва и стартира двете основни услуги едновременно.
*   `.gitignore` — Файл, указващ кои директории (напр. `node_modules`) да не се споделят в GitHub.

---

## 🛠️ Архитектура и компоненти

Проектът е разделен на две изолирани услуги (услуги/контейнери), които комуникират помежду си:

### 1. Web Application (`web-app`)
*   **Технология:** Node.js с рамката Express.js.
*   **Роля:** Приема клиентски заявки, извежда начална страница и осъществява връзка с базата данни за валидиране на свързаността.
*   **Контейнеризация:** Използва олекотен базов образ `node:18-alpine`. Порт `3000` в контейнера е пренасочен към порт `3000` на локалната машина.

### 2. Database (`database`)
*   **Технология:** MongoDB (официален образ `mongo:6.0`).
*   **Роля:** Служи като дълготрайно хранилище за данни (NoSQL DB).
*   **Контейнеризация:** Работи на стандартния си порт `27017`. Използва се Docker Volume за запазване на данните (Data Persistence).

---

## 🔄 Комуникация между услугите

Комуникацията се осъществява изцяло в изолираната вътрешна мрежа, която Docker Compose създава автоматично по подразбиране:
*   Вместо използването на статични IP адреси или `localhost`, уеб приложението се свързва към базата данни чрез **името на услугата** като хост.
*   Връзката е дефинирана чрез променливата на средата `MONGO_URI=mongodb://database:27017/my-docker-db`, където `database` е името на MongoDB контейнера.
*   Използвана е директивата `depends_on`, която гарантира, че базата данни ще стартира преди уеб приложението.

---

## 💾 Управление на данните (Data Persistence)

За да не се губят данните в базата данни при рестартиране или изтриване на контейнера, е конфигуриран именован дял (Named Volume):
*   **Име на дела:** `mongo-data`
*   **Път вътре в контейнера:** `/data/db` (където MongoDB съхранява файловете си).
